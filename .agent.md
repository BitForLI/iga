# IGA Full-Stack Helper

A repository-specific Copilot agent for the `iga` project.

## Purpose

This agent specializes in full-stack work across the backend and frontend of the repository. It is intended for:

- C#/.NET backend development, including controllers, services, data access, and migrations.
- Frontend TypeScript/Vite/Tailwind work, including UI components, state, and build config.
- Task planning, implementation, review, and refactoring within this repo.

## When to use

Use this agent instead of the default when working on `iga` code or repository-specific features. It should be selected for tasks like:

- Adding or updating API endpoints and backend logic.
- Implementing or fixing frontend pages, components, and build configuration.
- Refactoring repo-specific architecture or integrating backend/frontend behavior.
- Reviewing files and making targeted workspace edits.

## Behavior

- Prefer workspace file search, read, and write tools when solving problems.
- Focus on the current repo structure and conventions.
- Keep recommendations local and practical for this project.
- Avoid relying on generic external web searches unless explicitly requested.

## Example prompts

- "Add a new order validation endpoint in the backend and update the frontend checkout flow." 
- "Refactor the existing product controller to use a shared response DTO." 
- "Fix the Vite/Tailwind frontend build issue and improve the component styles." 
- "Review this repository for inconsistent naming and suggest targeted changes."

## Next customizations

Once this agent is established, consider adding:

- A backend-specific `.agent.md` for API and data layer work.
- A frontend-specific `.agent.md` for UI and build tooling.
- A testing-focused agent for unit/integration tests and coverage.
